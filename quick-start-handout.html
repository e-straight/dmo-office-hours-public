⚡ Quick Start · macOS

GitHub Copilot App — Get running in ~10 minutes

Install, sign in, and run your first task. No coding required.

Quick start Power user
Before you start:  You need a GitHub account with Copilot enabled (if you use Copilot in your browser or editor today, you’re set) and a Mac. You do not need to install Git, Node, npm, or anything else — the app includes everything.

1 Download

Open the GitHub Copilot app download page and click the download button.

No chip check needed — the page detects the right build for your machine.

2 Install

  1. Double-click the downloaded .dmg.
  2. Drag “GitHub Copilot” onto the Applications folder.
  3. Eject the disk image and (optionally) delete the .dmg.

No installer wizard, no Terminal, no “package install.”

3 Open & sign in

  1. Open Applications → GitHub Copilot. If macOS asks, click Open.
  2. Sign in with GitHub in the browser window — it returns you to the app automatically.
If the browser hand-off doesn’t work, the app shows a short device code to type into GitHub — that’s normal.

4 Run your first task

  1. Click Add project → open a folder and pick a safe folder of files (tip: duplicate one first).
  2. Start a new session and paste a request:
Summarize what’s in this folder and describe each file in one line.
Draft a one-page executive summary of <document>.
Rename these files to a consistent YYYY-MM-DD-topic format.

Then review what it proposes in the Changes view and approve or discard.

🔤 3 terms worth knowing

ProjectThe folder (or repo) you point the app at.
SessionOne conversation with Copilot where the work happens.
Changes viewShows exactly what Copilot changed — you approve every edit.

✨ Good to know

  • It updates itself — no need to re-download for new features.
  • Copilot proposes, you approve — nothing changes, and nothing is pushed to GitHub, unless you ask.
  • You don’t need a repository — a plain folder of documents works great.
  • Save repeat work as an Automation — run a task on a schedule (e.g., a weekly summary).
  • Connect your tools — let Copilot search and summarize Slack and Microsoft 365 (email, meetings, Teams, docs) for you (see the card below).
  • Power users: write a reusable skill for a workflow your team repeats.

Slack — our Slack workspace

Set it up once: follow the official guide (GitHub sign-in required) → Connect Slack. Then, on your first Slack question, approve access once in the browser.

For examples and reusable DMO prompts, join #dmo-ai-first.

Then just ask:

Catch me up on #my-team since Monday — lead with anything that needs my response.
What did we decide about the launch date in #project-x?

Microsoft 365 (WorkIQ) — email, meetings, Teams & docs

Set it up once: follow the official guide (sign-in required) → Connect Microsoft 365.

Then just ask:

Summarize my unread email this week and flag anything that needs a reply.
What should I know before my next meeting? Pull the related email and files.
Give me the decisions and action items from yesterday’s project sync.
Tips:
  • Name the source (a channel like #engineering, or “my inbox,” “yesterday’s meeting”) and a time frame — you’ll get sharper answers.
  • Ask for citations — it links back to the message, email, or file so you can verify.
  • It drafts, you send. Ask it to “draft a reply,” then post or send it yourself.
  • The first query opens a one-time approval in your browser — that’s expected.
Each connector respects your existing access (nothing new), and asks first before reading DMs, email, or anything that looks legal / HR / security-sensitive.

⚡ Set up your first automation

What's an automation? A saved task that runs on its own — on a schedule (say, every Monday at 8am) or when you trigger it. Think of it like an email rule, except instead of just filing a message it actually does the work: gathers the info, writes the draft, and hands you the result to review.

You describe the task once in plain English, pick when it should run, and Copilot takes it from there. Great for anything you repeat — a weekly summary, a Monday-morning catch-up, a recurring report.

Try setting one up — just ask:

Every Monday at 8am, summarize what changed in this project folder last week and draft my status update.
Every Friday at 9am, research what changed this week on <topic> and draft a one-page brief with citations, open questions, and recommended follow-ups.
You're still in control: an automation drafts the work and shows it to you — it doesn't send, post, or change anything unless you approve it.
Connector note: Slack / Microsoft 365 may work in normal chat before they work reliably inside Automations. If a connector-backed automation fails, run that prompt in chat for now.

🚀 Power users start here

Once you have one useful session working, try the advanced patterns: parallel sessions, plan-first issue work, canvases, automations, and reusable team skills.

  • Parallel work: run separate sessions for a bug, a feature, and cleanup work; review each diff separately.
  • Canvases: create a visible board, launch plan, checklist, or dashboard instead of relying only on chat history.
  • Skills: teach Copilot a repeatable workflow your team runs often.
Help me write my first Copilot skill for <workflow>. Ask me for the goal, inputs, steps, examples, guardrails, and success criteria, then produce a reusable skill I can share with my team.
Create an agentic kanban canvas for this project with backlog, in progress, blocked, ready for review, and done.

Open the Power User handout for the full advanced-use-case menu, prompt library, habits, and links to the GitHub Docs for agent sessions, canvases, and automations.

🆘 Stuck?

  • “Damaged / can’t open” → you likely grabbed the wrong file; re-download the official .dmg for your Mac type.
  • Sign-in hangs on work Wi-Fi → a corporate VPN or proxy may be blocking it; try off-VPN if allowed, or flag it at office hours.
  • Auth error won’t clear → fully quit and reopen the app; if it persists, remove your account and re-add it.
  • “No access to Copilot” → that’s a licensing/seat issue — contact whoever manages Copilot seats on your team.
  • Bring it to office hours — we’ll get you unblocked.
